## 2. Change Trigger Catalog

| Trigger | Example | Required Action |
|---------|---------|-----------------|
| Business strategy change | New product model | Reassess capability and roadmap |
| Regulatory/security change | New compliance requirement | Update controls and architecture artifacts |
| Technology risk signal | Critical dependency issue | Run architecture impact assessment |
| Delivery friction signal | Repeated boundary conflicts | Revisit service/domain boundaries |

## 3. Change Classification

| Type | Criteria | Governance Path |
|------|----------|-----------------|
| Minor | Localized, low-risk | Standard architecture review |
| Major | Cross-domain impact | Architecture board + ADR required |
| Strategic | Enterprise model shift | New vision cycle (Phase A refresh) |

## 4. Review Cadence

| Cadence | Scope |
|---------|-------|
| Sprint | Emerging deviations and tactical updates |
| Release | Compliance and traceability checks |
| Quarterly | Architecture health and roadmap alignment |
