# G02.03 Access and System Shell Support

**Track:** R01. AUTHENTICATION AND AUTHORIZATION, R02. TENANT MANAGEMENT  
**Technology:** PostgreSQL support tables plus runtime access contracts  
**Status:** MVP Review Ready  
**Supports:** `G01.01 Access to System Services`

---

## 1. Purpose

This document defines the **data-model and runtime-support layer** for:

- `G01.01.01 Login Scenarios`
- `G01.01.02 System Shell`

It covers the persisted support tables and runtime contracts needed for:

- login provider selection
- recovery-code handling
- rate-limit state
- provider-availability state
- tenant-resolution and tenant-access outcomes
- role-based home and shell navigation
- help links
- about/version/legal content
- notification feed and read state

This document does not replace:

- the tenant registry database, which remains the source for tenant identity and platform tenant state
- the tenant manager PostgreSQL database, which remains the source for tenant users, auth-provider registrations, sessions, MFA configuration, and tenant-facing notifications

## 2. Persisted Support Tables

### 2.1 Login Support Tables

#### 2.1.1 `user_recovery_code_sets`

| Column | Type | Key / Rule | Purpose |
|--------|------|------------|---------|
| `recovery_code_set_id` | `UUID` | PK | Recovery-code batch identifier |
| `user_id` | `UUID` | FK -> `tenant_users.user_id` | Parent user |
| `issued_at` | `TIMESTAMPTZ` | NOT NULL | Batch issue timestamp |
| `expires_at` | `TIMESTAMPTZ` | NULL | Optional expiry timestamp |
| `set_status` | `VARCHAR(20)` | NOT NULL | Current set status |

#### 2.1.2 `user_recovery_codes`

| Column | Type | Key / Rule | Purpose |
|--------|------|------------|---------|
| `recovery_code_id` | `UUID` | PK | Recovery-code identifier |
| `recovery_code_set_id` | `UUID` | FK -> `user_recovery_code_sets.recovery_code_set_id` | Parent code set |
| `code_hash` | `TEXT` | NOT NULL | Stored hash of the recovery code |
| `consumed_at` | `TIMESTAMPTZ` | NULL | Consumption timestamp |
| `active_flag` | `BOOLEAN` | NOT NULL | Code is still valid for use |

#### 2.1.3 `login_rate_limit_windows`

| Column | Type | Key / Rule | Purpose |
|--------|------|------------|---------|
| `rate_limit_window_id` | `UUID` | PK | Rate-limit window identifier |
| `tenant_url` | `VARCHAR(255)` | NOT NULL | Target tenant entry point |
| `subject_key` | `VARCHAR(255)` | NOT NULL | User, identifier, IP, or composite rate-limit subject |
| `provider_family_key` | `VARCHAR(100)` | NULL | Target auth-provider family when applicable |
| `attempt_count` | `INTEGER` | NOT NULL | Attempts observed inside the current window |
| `window_started_at` | `TIMESTAMPTZ` | NOT NULL | Window start timestamp |
| `window_ends_at` | `TIMESTAMPTZ` | NOT NULL | Window end timestamp |
| `blocked_until` | `TIMESTAMPTZ` | NULL | Temporary block expiry |
| `limit_status` | `VARCHAR(20)` | NOT NULL | Open, warning, blocked, or expired state |

#### 2.1.4 `auth_provider_runtime_status`

| Column | Type | Key / Rule | Purpose |
|--------|------|------------|---------|
| `provider_runtime_status_id` | `UUID` | PK | Runtime-status identifier |

| `provider_key` | `VARCHAR(255)` | NOT NULL | Runtime provider key |

## 3. Runtime Access Contracts

Some business outcomes in `G01.01` are runtime evaluations, not persisted business rows. They still require explicit support contracts.

### 3.1 `TenantResolutionOutcome`

| Field | Type | Purpose |
|-------|------|---------|
| `tenant_url` | `string` | Requested tenant URL |
| `resolution_status` | `string` | `RESOLVED`, `DELETED`, `NOT_FOUND` |
| `tenant_ref` | `string` | Resolved tenant identifier when found |
| `resolved_at` | `timestamp` | Resolution timestamp |

This contract supports:

- `TP04-V01 Tenant Deleted or Not Exist - 404`

### 3.2 `LoginAccessEvaluation`

| Field | Type | Purpose |
|-------|------|---------|
| `tenant_ref` | `string` | Resolved tenant identifier |
| `tenant_status` | `string` | Current tenant status from tenant registry |
| `license_state` | `string` | No license, active, expired, or similar evaluated state |
| `seat_state` | `string` | Active seat, no active seat, admin-only override, or similar result |
| `provider_availability_state` | `string` | Available, degraded, unavailable, or unknown |
| `rate_limit_state` | `string` | Open, warning, blocked, or cleared |
| `allowed_actions` | `string[]` | Allowed next actions for the current outcome |
| `evaluated_at` | `timestamp` | Evaluation timestamp |

This contract supports:

- `TP01-V06 No Tenant License - Default Admin Only`
- `TP01-V07 Tenant License Expired - Default Admin Only`
- `TP01-V08 Suspended Tenant - Access Blocked`
- `TP02-V08 No Active Seat Error`
- `TP02-V09 Rate-Limited Error`
- `TP02-V10 Auth Provider Unavailable Error`

### 3.3 `MfaChallengeContract`

| Field | Type | Purpose |
|-------|------|---------|
| `user_id` | `string` | Challenged user reference |
| `challenge_state` | `string` | Pending, failed, expired, or complete |
| `default_method` | `string` | TOTP or other configured method |
| `recovery_code_allowed` | `boolean` | Recovery-code fallback is permitted |
| `expires_at` | `timestamp` | Challenge expiry |

This contract supports:

- `TP03-V01 TOTP Code Entry`
- `TP03-V02 Invalid MFA Code Error`
- `TP03-V03 Recovery Code Entry`
- `TP03-V04 MFA Session Expired`

### 3.4 `ShellNotificationFeed`

| Field | Type | Purpose |
|-------|------|---------|
| `user_id` | `string` | Current user |
| `notification_count` | `integer` | Total notifications in the current feed |
| `unread_count` | `integer` | Unread notification count |
| `items` | `list` | Notification feed items with title, severity, route, and read state |
| `loaded_at` | `timestamp` | Feed load timestamp |

This contract supports:

- `TP05-V01 Notification Loading`
- `TP05-V02 Notification List`
- `TP05-V03 Notification Empty`
- `TP05-V04 Notification Error`
